# Code Of Conduct

[简体中文](docs/CODE_OF_CONDUCT.zh-CN.md)

This repository aims to keep discussions direct, professional, and workable.

## Expected Behavior

- Discuss issues using facts, evidence, and reproducible information
- Respect different experience levels, technical approaches, and ways of communicating
- Give clear, actionable feedback on code, documentation, and process
- Clarify misunderstandings before escalating conflict
- Show basic patience to new contributors, especially on their first issue or pull request

## Unacceptable Behavior

- Personal attacks, humiliation, discrimination, harassment, or threats
- Repeated provocation, spam, or bad-faith argument intended to create conflict
- Publicly exposing private information, credentials, internal links, or other sensitive data
- Posting unrelated advertising, phishing, or other spam in issues, pull requests, or comments
- Knowingly misrepresenting facts, faking validation results, or repeatedly ignoring repository rules

## Scope

This code of conduct applies to issues, pull requests, code review, commit-related discussion, and other public interactions directly tied to this repository.

## Enforcement

- Maintainers may remove inappropriate comments, issues, pull requests, or related discussion
- Repeated or severe violations may lead to interaction limits, closed discussions, or rejected contributions
- Enforcement prioritizes protecting the collaboration environment and reducing ongoing disruption

## Reporting

- If you encounter a conduct issue, contact the maintainers with the relevant context
- If GitHub public discussion is your only contact path, do not post sensitive details publicly; only note that private follow-up is needed
- Do not use this document for security vulnerabilities, credential leaks, or other sensitive security issues; use [SECURITY.md](SECURITY.md) instead

## Notes

This is not legal text and does not try to enumerate every scenario.
For cases not explicitly listed here, maintainers will judge based on whether the behavior disrupts collaboration, adds avoidable cost to others, or harms sustainable maintenance of the project.
