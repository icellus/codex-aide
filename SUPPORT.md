# Support

[简体中文](docs/SUPPORT.zh-CN.md)

## Good Issue Candidates

- reproducible defects, regressions, or documentation errors in `codex-aide`
- clear inconsistencies between rules, scripts, templates, or documented behavior
- stable post-release behavior problems that can be reproduced after installation
- concrete feature requests or governance/process improvement proposals

## Not Good Issue Candidates

- business-code problems unrelated to this repository
- environment, permission, CI, or deployment failures owned by a downstream repository
- vague complaints with no basic context or actionable scope
- security vulnerabilities or credential leaks

For security matters, use [SECURITY.md](SECURITY.md) instead.

## What To Prepare First

- the file paths, commands, or entrypoints involved
- expected behavior versus actual behavior
- minimal reproduction steps
- relevant logs, errors, or validation output
- impact scope: this repository only, downstream repositories only, or both

## Recommended Paths

- defects: GitHub Issues using the bug template
- feature requests: GitHub Issues using the feature template
- ready-to-review changes: open a pull request and fill in the validation section

## Response Boundaries

- This repository is maintained on a best-effort basis and does not promise a fixed SLA
- Reproducible issues with clear scope and strong evidence are prioritized
- For under-specified reports, maintainers may ask for more context before deciding whether to continue
