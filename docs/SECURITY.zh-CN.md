[English](../SECURITY.md)

## 支持范围

当前维护范围仅包括本仓库中的 `codex-aide` 及其配套维护脚本、规则数据和文档。

以下情况不在本仓库安全响应范围内：

- 已迁出的 `claude-starter`
- 你安装 `codex-aide` 后，目标仓库自身新增的业务代码、部署配置或第三方服务
- 明显属于上游平台、运行环境或 GitHub 自身设置的问题

## 如何报告安全问题

请不要公开提交包含可利用细节的 issue、PR 或评论。

建议按以下顺序处理：

1. 若仓库已启用 GitHub Private Vulnerability Reporting，优先使用该私密通道
2. 若你与维护者已有私下联系方式，直接私下发送报告
3. 若暂时只有公开入口，请先创建不含细节的最小说明，请求维护者转入私下沟通

报告中建议包含：

- 影响范围
- 复现步骤或触发条件
- 可能的利用方式
- 暂时规避方案（如果有）
- 你期望的公开披露节奏（如果有）

## 响应预期

- 维护者会尽量在 7 天内确认是否收到报告
- 若问题成立，会尽量给出处理状态、影响边界和修复计划
- 若暂时无法快速修复，也会说明当前风险和建议的临时缓解方式

## 披露原则

- 在修复、缓解或确认不受影响之前，不建议公开披露可直接利用的细节
- 修复发布后，可在合适时机补充公开说明或变更记录

## 普通问题入口

非安全问题请使用常规协作入口：

- 使用问题或功能建议：见 [SUPPORT.zh-CN.md](SUPPORT.zh-CN.md)
- 一般贡献流程：见 [CONTRIBUTING.zh-CN.md](CONTRIBUTING.zh-CN.md)
